Automation Builder & Executive Operations

Fully automated weekly executive summary system that pulls Microsoft 365 email and calendar data, processes it through Claude AI, and delivers a structured HTML briefing to the executive inbox every Monday at 8AM — with zero manual inputs from anyone on the team.

How it was built

  • Connected the M365 mailbox and calendar via Microsoft Graph API through a Zapier multi-step automation triggered every Sunday night
  • Digest data routed through a Claude API prompt engineered to extract action items, key decisions, and priority threads — then formatted as a clean HTML email
  • Power Automate handles the final delivery step, pushing the briefing at exactly 8AM Monday to the correct exec inbox
  • Edge cases handled: empty week detection, deduplication of recurring calendar entries, and fallback formatting if Claude response exceeds token limits
  • Covers reporting for 70+ locations — no configuration changes needed as new stores are added

Document Classification & Routing System

Systems Automation Builder

End-to-end document classification and routing system built for a 70+ location multi-unit operator group. Zapier-based automation reads incoming files, classifies them across 8 document categories using AI, and routes each file to the correct SharePoint library subfolder — eliminating a 2–3 hour/week manual filing task entirely.

How it was built

  • Trigger: Zapier watches a shared email inbox and a SharePoint upload folder — any new attachment fires the automation
  • Classification: file name, sender domain, and extracted text are passed to an AI prompt that returns one of 8 categories: invoices, leases, contracts, permits, compliance, HR, marketing, or ops
  • Routing logic extracts a market or location identifier from the filename convention or sender metadata, then maps it to the correct SharePoint subfolder path
  • Unclassified files (confidence below threshold) are routed to a review queue with a Teams notification — never silently dropped
  • Scales to new locations automatically: adding a market only requires updating the location lookup table, not rebuilding any Zap logic
